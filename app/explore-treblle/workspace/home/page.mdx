# Home

The **Home dashboard** provides a high-level overview of your entire API ecosystem. It aggregates key operational, performance, and governance metrics into a single view so you can quickly understand the health of your APIs and the activity across your environments.

From this page, teams can monitor uptime, request volume, endpoint usage, integrations, and customer activity without navigating through individual APIs.

<ImageCard src="/og-images/Home - overview.png" />

## Dashboard Overview

At the top of the Home dashboard, Treblle displays a **summary of the current API health and activity across your workspace**.

This section provides a quick snapshot of key operational metrics.

### APIs Health

Shows the overall health status of your APIs. When all monitored services are operating normally, the system displays a healthy status indicator.

### Uptime (Last 24h)

Displays the uptime percentage for your APIs over the last 24 hours. This allows teams to quickly assess service availability and reliability.

### Total APIs

The total number of APIs currently connected and monitored within the workspace.

### Total Requests

The total number of API requests processed across all APIs during the selected time range.

### Total Endpoints

Shows the number of endpoints currently observed across all APIs.

### Total Customers

Represents the number of unique consumers or integrations interacting with your APIs.

## Navigation Tabs

The Home dashboard allows you to explore insights across several operational perspectives using navigation tabs.

### Overview

Provides a broad operational summary across all APIs, including request activity, integrations, categories, and usage patterns.

### Security

Displays security-related insights such as authentication patterns, suspicious requests, and policy violations.

### Performance

Focuses on request latency, throughput, and response performance across APIs and endpoints.

### Governance

Provides insights related to API ownership, categorization, and policy management.

### Customers

Shows activity and usage patterns related to API consumers.

### Agents

Displays insights generated by Treblle AI agents responsible for monitoring and analysis.

---

## Filters

The Home dashboard includes filters that allow you to adjust the scope of the data displayed.

You can filter metrics based on:

### Environment

View data for specific environments such as production, staging, or development.

### Category

Filter APIs based on categories or internal classifications.

### Time Range

Adjust the timeframe of displayed metrics, such as the last 24 hours.

---

## API Activity Summary

This section highlights key operational indicators across your APIs.

### APIs

Displays the total number of APIs and the percentage change over time.

### Requests

Shows the total number of API requests processed within the selected timeframe.

### Endpoints

Displays the number of active endpoints currently being monitored.

### Customers

Shows the number of API consumers interacting with your APIs.

These metrics provide a quick operational snapshot that helps teams understand growth and usage trends.

## API Type Distribution

The **API Type** chart visualizes how APIs are distributed across different exposure models.

Typical categories include:

- Internal APIs
- External APIs
- Partner APIs

This distribution helps teams understand how APIs are being used across internal systems, partners, and external developers.

---

## API Integrations Across Environment

This visualization shows how APIs are integrated across various infrastructure platforms and environments.

Examples may include:

- MuleSoft
- AWS API Gateway
- Azure API Management
- Framework-based APIs such as Django or Laravel

The chart also highlights how APIs are distributed across:

- UAT
- Development
- Production

This allows platform teams to understand how integrations evolve across environments.

## New Problems

The **New Problems** widget highlights newly detected issues across the API ecosystem.

These issues may include:

- Error spikes
- Failed requests
- Performance anomalies
- Integration disruptions

This section helps teams quickly detect and respond to emerging operational issues.

---

## CO₂ Emissions

Treblle provides an estimated **CO₂ emissions metric** based on API traffic and infrastructure activity.

This metric helps organizations monitor the environmental impact of their API operations and track improvements over time.

- API name

- Performance trend indicators

This allows teams to quickly identify high-traffic services and prioritize monitoring or optimization efforts.

This provides visibility into the underlying infrastructure powering the API ecosystem.
