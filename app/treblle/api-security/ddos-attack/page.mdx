# DDoS Attack Detection

Treblle's DDoS Attack Detection feature helps you identify potential Distributed Denial of Service (DDoS) attacks by monitoring traffic patterns and flagging unusual request spikes that could indicate malicious activity.

## How DDoS Detection Works

Treblle continuously monitors your API traffic and compares current request volumes to historical patterns to identify potential DDoS attacks.

The system tracks request spikes by:

### Threat Level Classification

DDoS threats are categorized into three levels based on traffic increase percentages:

<InfoTable title="Threat Level" description="Details">
    <InfoTable.Row
        title="Low Threat (80-150% increase)"
        description="Moderate traffic spike that could be legitimate increased usage or a small-scale attack. Monitor closely and investigate if sustained."
    />
    <InfoTable.Row
        title="Medium Threat (150-350% increase)"
        description="Significant traffic spike likely indicating coordinated activity. Implement rate limiting and investigate source IPs."
    />
    <InfoTable.Row
        title="High Threat (350%+ increase)"
        description="Severe traffic spike strongly suggesting a DDoS attack. Immediate action required - activate DDoS mitigation measures."
    />
</InfoTable>

## Accessing DDoS Monitoring

### Enable DDoS Monitoring Widget

1. **Navigate to API Dashboard**
   
   Go to your individual API dashboard for the API you want to monitor.

2. **Open Customize Dashboard**
   
   Click the **Customize Dashboard** button (four squares icon) on the right side of the dashboard.

   In the customize menu, find and enable the **"Denial of Service"** widget.

   <ImageCard src="/platform/api-security/sub/enable-ddos.jpg" alt="Enable DDoS Monitoring Widget" />

3. **Save Changes**
   
   Click **Save Changes** to add the DDoS monitoring widget to your dashboard.

## Preventive Measures

<TipCard>
Implement multiple layers of protection to effectively defend against DDoS attacks.
</TipCard>

To protect against DDoS attacks:

<Checklist>
    <Checklist.item id="rate-limiting">**Configure Rate Limiting** - Set appropriate request limits per IP and implement progressive delays for repeated requests</Checklist.item>
    <Checklist.item id="cdn-services">**Use CDN Services** - Leverage Content Delivery Networks for traffic distribution and enable DDoS protection features</Checklist.item>
    <Checklist.item id="ip-monitoring">**Monitor IP Reputation** - Use Treblle's <Link href="/treblle/api-security/ip-address/">Risky IPs</Link> feature and implement automatic blocking</Checklist.item>
    <Checklist.item id="custom-alerts">**Set Up Custom Alerts** - Create <Link href="/features/custom-alerts/">Custom Alerts</Link> for traffic spikes and configure immediate notifications</Checklist.item>
</Checklist>
